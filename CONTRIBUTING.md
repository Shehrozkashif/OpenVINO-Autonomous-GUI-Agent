## How Can I Contribute?

### Reporting Bugs
* **Check the FAQ** to see if your issue has already been addressed.
* **Search existing issues** to see if the bug has already been reported.
* If you find a new bug, please **open a new issue**, providing as much detail as possible (OS, hardware, logs, reproduction steps).

### Suggesting Enhancements
* Open an issue with the tag `enhancement`.
* Provide a clear and concise description of the proposed feature.
* Explain why this enhancement would be useful to most users.

### Pull Requests
1.  **Fork the repository** and create your branch from `main`.
2.  **Ensure the test suite passes** by running `pytest`.
3.  **Follow the existing code style**: We use PEP 8 and standard Python type hinting.
4.  **Document your changes**: Update the README or add docstrings if necessary.
5.  **Issue a Pull Request** with a detailed description of your changes.
